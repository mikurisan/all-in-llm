混合检索 (Hybrid Search) 结合了稀疏向量 (Sparse Vectors) 和密集向量 (Dense Vectors) 的优势.

## 1 稀疏 VS 密集

### 1.1 稀疏向量

也称为“词法向量”, 是基于词频统计的传统信息检索方法. 其通常是一个维度极高(与词汇表大小相当), 但绝大多数维度为 0 的 vector. 其将文档视为 word 的 collection, 不考虑语义, 每个维度对应一个 word, 非 0 的维度代表该词在文档中的权重.

先根据所有文档计算出每个 word 的权重, 然后就可以给出各个文档的稀疏向量表示, 随后将 query 也转换为稀疏向量表示, 与各个文档向量计算得到 similarity.

该方法无需训练, 可解释性强, 能够实现关键词匹配, 对于专业名词和特定名词检索效果很好. 但是无法理解语义, 识别同义词.

### 1.2 密集向量

也称“语义向量”, 是通过深度学习模型得到的低维的, 稠密的浮点数表示. 在理想的语义空间中, vector 之间的 distance 代表了其语义之间的关系.

一个经典的例子是 `vector('国王') - vector('男人') + vector('女人')` 的计算结果在向量空间中非常接近 `vector('女王')`.

该方法能够理解同义词, 近义词和上下文关系, 泛化能力强. 但可解释性差, 需要大量的资源进行训练.

> 对于未登录词 (OOV, Out-of-Vocabulary), 传统的稀疏向量会完全忽略, 而现代的密集向量能通过子词分割来处理.

## 2 混合检索

### 2.1 技术原理与融合方法

#### 2.1.1 倒数排序融合(Reciprocal Rank Fusion, RRF)

只关心每个 document 在其结果集中的 rank, rank 越靠前最终得分越高.

#### 2.1.2 加权线性组合

将不同检索系统的得分归一化, 然后通过总和为 1 的权重系数进行线性组合. 通过调整权重可以控制对应检索系统在最终排序得分中的权重.

比如电商系统中可以调高 sparse vector 的权重.

### 2.2 优势与局限

| 优势	| 局限 |
| -- | -- |
| 召回率与准确率高: 能同时捕获关键词和语义, 显著优于单一检索. |	计算资源消耗大: 需要同时维护和查询两套索引. |
| 灵活性强: 可通过融合策略和权重调整, 适应不同业务场景. |	参数调试复杂: 融合权重等超参数需要反复实验调优. |
| 容错性好: 关键词检索可部分弥补向量模型对拼写错误或罕见词的敏感性 |	可解释性仍是挑战: 融合后的结果排序理由难以直观分析. |
